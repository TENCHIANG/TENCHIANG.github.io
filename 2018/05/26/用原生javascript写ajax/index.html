原生ajax请求写法

XMLHttpRequest对象的属性

onreadystatechange

存储函数或函数名，每当readyState属性改变时，就会调用该函数。

readyState

存有XMLHttpRequest的状态类，0-4的取值范围。

  • 0：初始化，XMLHttpRequest对象还没有完成初始化
  • 1：载入，XMLHttpRequest对象开始发送请求
  • 2：载入完成，XMLHttpRequest对象的请求发送完成，已收到全部响应内容但尚未解析
  • 3：解析，XMLHttpRequest对象开始解析服务器的响应内容
  • 4：完成，XMLHttpRequest对象读取服务器响应结束

    status

  • 200: OK
  • 404: 未找到页面

数组与对象的合并

数组

  • concat

  • push

  • apply

方法名称 返回值 是否改变原值 是否平行合并
concat 合并后新的数组 否 是
push 合并后元素个数 是 否
apply 合并后元素个数 是 是

对象

  • assign

  • \$.extend

上面看起来assign与\$.extend没有任何区别，都可接收多个参数、都可用空对象实现不改变原值，\$.extend还需要jQuery或者Zepto的支持，那\$.extend存在的意义是啥？答案是\$.extend可以对对象进行深拷贝。

浅拷贝与深拷贝

浅拷贝

深拷贝

如果在单层次的对象里，使用浅拷贝就足够了，但是如果有多层次的对象，浅拷贝只会拷贝引用，就形成了依赖，就得考虑使用深拷贝。另外数组都是深拷贝。
